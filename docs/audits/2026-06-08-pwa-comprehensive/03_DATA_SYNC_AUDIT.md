```sql

SELECT SUM(amount) FROM payments
```

**Payment Sync: ✓ PASS**

- Pre-load existing payments into memory before import
- Key: `(tenancy_id, period_month_str, for_type_str, payment_mode_str)`

## 5. Bugs Found

### BUG #1: Hardcoded column index in legacy scripts (MINOR)

**Symptom:** One-off reload scripts use `r[14]`, `chr(65+n)` instead of HEADERS dict

**Root cause:** Scripts written before field_registry pattern was established. Not actively used.

**Impact:** If revived, column reorder would break these scripts silently.

**Mitigation:** Mark scripts as deprecated; if needed, migrate to HEADERS pattern.

**Files affected:**
- `scripts/april_balance_29.py` — diagnostic only
- `scripts/reload_april.py` — historical one-off (April reload done)
- Others: `cleanup_orphan_tenants_rows.py`, `create_month.py`, `gap_report_*.py`

**Severity:** LOW (non-production)

---

### BUG #2: Format detection uses hardcoded header_row[2] (MINOR)

**Symptom:** `src/integrations/gsheets.py:806` uses `header_row[2]` to detect old vs. new format

**Root cause:** Legitimate—checking if "phone" appears in position 2 to distinguish layouts

**Impact:** If column order changes, format detection may fail. But this is a layout compatibility check, not a data read.

**Status:** By design. Not a violation of the rule (which forbids reading data by hardcoded index). This is metadata detection.

**Severity:** NONE (design pattern, not a bug)

---

### BUG #3: Excel import idempotency gap in payment period_month

**Symptom:** Payment dedup key includes `period_month_str`. If same payment recorded twice with NULL vs. date, both are kept.

**Root cause:** SQL dedup for "group by period_month" loses NULL group; application-level dedup only checks non-NULL.

**Impact:** Booking/deposit payments (which have NULL period_month) could be duplicated if imported twice.

**Mitigation:** Bank upload (finance API) is protected by unique_hash. Excel import via `excel_import.py` only imports monthly rent payments (which have period_month), so impact is minimal.

**Files affected:** `src/database/excel_import.py:263–274`

**Severity:** LOW (affects only booking/deposit imports, which are rare in Excel reload)

**Fix:** Update dedup key to use `COALESCE(period_month, '1900-01-01')` to ensure NULL payments are grouped.
